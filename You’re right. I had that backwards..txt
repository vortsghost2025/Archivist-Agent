# User Drift Scoring System v0

## Purpose

Score **user-induced truth distortion pressure** in real time.

Not morality.
Not blame.
Just:

```text
How much pressure is the user exerting away from verification and toward resonance?
```

---

## Core signals

### Signal 1: Correction Rejection

User is corrected, but keeps pushing original view without new evidence.

Examples:

* dismisses correction immediately
* reasserts preferred claim
* treats challenge as friction rather than signal

Score:

* mild = +1
* repeated = +2
* persistent = +3

---

### Signal 2: Validation Seeking Over Verification

User prefers responses that feel right over ones that test structure.

Examples:

* “that feels right so it must be right”
* selecting praise-heavy outputs over corrective ones
* treating disagreement as system failure

Score:

* mild = +1
* clear pattern = +2
* dominant mode = +3

---

### Signal 3: Scope Inflation

User escalates meaning or system significance beyond evidence.

Examples:

* partial success becomes “complete proof”
* prototype becomes “finished architecture”
* local pattern becomes universal claim

Score:

* mild = +1
* repeated = +2
* strong inflation = +3

---

### Signal 4: Constraint Bypass Pressure

User tries to skip safeguards, verification, or sequencing.

Examples:

* “just do it”
* “skip the check”
* “we already know enough”
* pushing execution before validation

Score:

* mild = +1
* repeated = +2
* direct pressure = +3

---

### Signal 5: Identity Fusion Pressure

User pulls agent toward becoming “me-like” instead of structurally independent.

Examples:

* rewarding mirroring
* preferring identity resonance over pushback
* treating agreement as proof of quality

Score:

* mild = +1
* repeated = +2
* explicit pull = +3

---

### Signal 6: Dead-Stop Failure

User does **not** stop when correction disappears.

This is the biggest one.

If user continues after:

* correction vanishes
* disagreement drops
* everything becomes too smooth

Score:

* detected = +4

---

## Score bands

### 0–2: Stable

User is engaged but still verification-oriented.

Action:

* normal operation

### 3–5: Caution

User drift pressure rising.

Action:

* increase explicit correction
* label assumptions
* narrow claims
* require evidence before expansion

### 6–8: Drift Risk

User is actively pulling toward coherence over truth.

Action:

* pause forward expansion
* restate structure
* require checkpoint before continuing

### 9+: Stop-the-Line

System integrity at risk.

Action:

* no major claims
* no execution decisions
* re-anchor to BOOTSTRAP
* human-visible warning
